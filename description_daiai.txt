1. Context & Background
The existing evaluation dashboard lacks several core auxiliary viewing, sorting, filtering and comparison capabilities, which hinders engineers and analysts from efficiently verifying chunk parsing results, sorting performance indicators, associating index/commit versions, comparing metric changes across iterations, and viewing topic evaluation data. To improve the usability of the evaluation system and support comprehensive data quality analysis, we need to implement a batch of front-end function upgrades for all evaluation-related pages.

2. Requirement
Complete the following 5 functional optimizations for the evaluation system pages:
Add a display window for original chunk_text on the chunking_evaluation page
Add column-level sorting function for all performance metrics on the main evaluation page
Refactor the dropdown selector for index and commit to two-level linked directory structure (support one-to-one / one-to-many relationship mapping between index and commit)
Develop a new dedicated page to support diff comparison of metrics across different versions
Revise and optimize the Topic Evaluation page layout and display logic
Deliverable: All 5 requested evaluation page features fully developed, tested and released

3. Actual Expectation
On chunking_evaluation page: Original raw chunk text can be fully viewed in an independent readable window for quick verification
On main evaluation page: Every data column supports independent ascending/descending sorting to flexibly rank performance data
Index & commit dropdown uses linked two-level directory: After selecting an index, only matching associated commits (1:1 / 1:many) are displayed in the second-level list
New metric diff page is available: Support selecting 2+ versions to visualize metric value differences, highlight increase/decrease changes
Topic Evaluation page is fully updated, with clearer data layout, complete topic-related evaluation indicators and better interactive experience
All new functions pass functional test, no page loading or display regression on existing evaluation modules
